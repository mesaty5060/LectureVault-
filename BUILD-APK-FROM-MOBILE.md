# 📱 طريقة بناء APK من الموبايل (GitHub Actions)

دا الدليل الكامل لبناء تطبيق LectureVault كـ APK جاهز للتثبيت، **بدون ما تثبت أي حاجة على الكمبيوتر**، كل حاجة من الموبايل أو المتصفح.

---

## 🎯 الفكرة باختصار

- ترفع المشروع على GitHub (مرة واحدة بس)
- GitHub بيبنى الـ APK تلقائياً على سيرفراته في ~10 دقايق
- بتنزّل الـ APK من تبويب Actions على موبايلك
- بتثبّته زي أي تطبيق

**مجاناً تماماً.**

---

## 📋 الخطوات

### الخطوة 1: عمل حساب GitHub (لو معندكش)

1. ادخل على [github.com](https://github.com) من المتصفح
2. اضغط **Sign up** وكمّل التسجيل (إيميل + باسورد)
3. أكّد الإيميل

### الخطوة 2: عمل Repository جديد

1. بعد ما تسجل دخول، اضغط على **+** أعلى يمين الشاشة → **New repository**
2. اكتب اسم الـ repo: مثلاً `LectureVault`
3. خليه **Public** (عشان GitHub Actions تشتغل مجاناً) أو **Private** برضو يشتغل
4. **متعملش** initialize بـ README (لإننا هنرفع ملفات جاهزة)
5. اضغط **Create repository**

### الخطوة 3: رفع ملفات المشروع

عندك طريقتين:

#### الطريقة (أ) — من المتصفح مباشرة (أسهل من الموبايل):

1. في صفحة الـ repo الجديد، اضغط **uploading an existing file** (الرابط الأزرق)
2. على الموبايل: اضغط **choose your files** أو السحب
3. **مهم:** لازم ترفع كل المجلدات مش بس الملفات. أسهل طريقة:
   - فك ضغط الـ ZIP على الموبايل (تطبيق ZArchiver أو RAR)
   - افتح GitHub في المتصفح ودوس **Add file → Upload files**
   - اختار كل محتويات مجلد `LectureVault` (مش المجلد نفسه)
4. اكتب commit message: `Initial commit`
5. اضغط **Commit changes**

#### الطريقة (ب) — تطبيق GitHub الرسمي:

1. نزّل تطبيق **GitHub** من Play Store أو App Store
2. سجّل دخول
3. للأسف تطبيق GitHub الرسمي مبيدعمش رفع ملفات مباشرة، فاستخدم الطريقة (أ) من متصفح الموبايل

#### الطريقة (ج) — استخدم تطبيق **Acode** أو **Termux** لو عايز git حقيقي:

```bash
git clone https://github.com/USERNAME/LectureVault.git
# انقل الملفات
git add .
git commit -m "Initial commit"
git push
```

### الخطوة 4: البناء يبدأ تلقائياً ✨

اللحظة اللي بترفع فيها الملفات، GitHub Actions بيبدأ شغل:

1. ادخل تبويب **Actions** في الـ repo
2. هتلاقي شغل بيتنفّذ اسمه **Build LectureVault APK**
3. استنى حوالي **8-12 دقيقة** أول مرة (المرات اللي بعدها أسرع بسبب الـ cache)
4. لما تخلص، هتلاقي ✅ خضرا

### الخطوة 5: تنزيل الـ APK

1. اضغط على الـ workflow run اللي خلص (✅ الخضرا)
2. انزل لتحت لحد ما تلاقي قسم **Artifacts**
3. اضغط على **LectureVault-debug-apk** — هينزل ZIP فيه الـ APK
4. فك ضغط الـ ZIP — هتلاقي `LectureVault-debug.apk`

### الخطوة 6: تثبيت الـ APK على موبايلك

1. افتح ملف الـ APK من مدير الملفات
2. هيقولك "Install from unknown sources" — وافق وفعّل الإذن
3. اضغط **Install**
4. التطبيق جاهز! 🎉

---

## 🔄 لو عدّلت في الكود

أي تعديل ترفعه على الـ repo (push)، GitHub Actions بتبني APK جديد تلقائياً. روح Actions → اختار آخر run → نزّل الـ artifact الجديد.

أو ابني يدوياً:
- Actions → **Build LectureVault APK** → **Run workflow**

---

## ⚠️ ملاحظات مهمة

- الـ APK ده **debug build** — مش موقّع توقيع رسمي، يعني للتجربة الشخصية فقط، مش للنشر على Play Store
- لو عايز Release APK موقّع، محتاج تعمل keystore وتضيفه كـ secret في GitHub — قولي وهجهزلك workflow تاني
- أول build بياخد وقت لأنه بينزّل dependencies (~500MB) ويعمل cache. اللي بعده ~5 دقايق
- الـ free tier في GitHub بيدّيك 2000 دقيقة بناء شهرياً — كفاية لمئات الـ builds
